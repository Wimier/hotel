# 🏨 易宿酒店在线预订与管理系统 (Hotel Booking Full-Stack System)

## 📝 项目简介

本项目是一款基于 **SpringBoot** 和 **Uni-app** 开发的全栈酒店预订系统。系统分为**移动端小程序**（面向普通用户）和 **PC 端后台管理系统**（面向酒店商户及系统管理员）。

系统不仅实现了基础的酒店查询、预约下单和订单查看功能，还增加了评价系统以及地图定位查找酒店功能，且深度适配了多商户经营场景，实现了严密的数据权限隔离机制。

---

## 🚀 技术架构

### 后端技术栈 (hotel-backend)

* **核心框架**：Spring Boot(基于 RuoYi 框架二次开发)
* **权限安全**：Spring Security + JWT 
* **持久层**：MyBatis + MySQL 8.0
* **开发工具**：IntelliJ IDEA, Maven

### 前端技术栈 (hotel-frontend)

* **用户端**：Uni-app + Vue 2 (支持发布为微信小程序/H5)
* **管理端**：Vue 2 + Element UI
* **开发工具**：HBuilderX, VS Code

## ✨ 核心功能亮点

### 1. 角色数据隔离

* **逻辑实现**：通过后端的 `merchant_id` 字段与登录用户的 `UserId` 进行动态绑定。
* **身份分流**：系统能自动识别登录者身份：
* **商户角色**：开启数据过滤，仅能管理自身名下的酒店及订单。
* **普通用户/管理员**：支持查看全平台酒店，满足 C 端用户的预订需求。

### 2. 完整的订单与评价功能

* **业务闭环**：支持从选房、下单、入住状态控制到售后评价的全流程管理。
* **状态机驱动**：订单状态实时同步，商户可一键操作“代客下单”或“详情修改”。
* **商家回复评价**:商家可以在后台对用户评价进行回复。
* **匿名评价机制**：订单评价支持匿名功能，保障用户隐私。

### 3. 腾讯地图 SDK 集成与 LBS 地理位置

* **实时定位与城市感知**：集成腾讯地图 SDK，系统启动后可自动触发 `uni.getLocation` 获取当前地理位置，精准识别用户所在城市，为后续的酒店筛选提供地域上下文支持。
* **交互式地图选点**：在商户端新增或修改酒店信息时，支持可视化地图选点功能。商户通过拖动标记点即可实时抓取 `longitude`（经度）与 `latitude`（纬度）高精度坐标，并同步至数据库。
* **自动化逆地址解析**：系统建立坐标与物理地址的联动机制，能自动将地图选出的坐标点转化为结构化的 `detail_address`（详细地址）并自动填充表单，极大提升了入驻效率并确保了地理数据的准确性。
* **一键外部地图导航**：利用 `uni.openLocation` 接口，系统支持将酒店坐标点传递至系统底层地图服务。

## 🛠️ 快速启动指南

### 1. 后端部署

1. 创建 MySQL 数据库并导入项目中的 `.sql` 脚本。
2. 修改 `application-druid.yml` 中的数据库连接信息。
3. 启动本地redis。
4. 启动 `RuoYiApplication.java`及后台`npm run dev `。

### 2. 前端部署

1. 使用 **HBuilderX** 打开 `hotel-frontend` 文件夹。
2. 修改 `request.js` 中的 `baseUrl` 为你本地的后端地址（如 `http://localhost:8080`）。
3. 点击“运行”编译到内置浏览器或微信开发者工具。
